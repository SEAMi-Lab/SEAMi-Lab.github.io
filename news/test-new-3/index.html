# markdown-it-template

为Markdown编写的的模板插件，可以自定义插件实现的具体功能。

默认模板格式如下：{{name|some words ....|config1|config2=xxx|config3=a,b,c,...}}

TIP

模板之间可以嵌套

# 安装

# API

js
var md = require('markdown-it')()
            .use(require('markdown-it-template'), name [, options]);

# 基础用法

使用该插件必须要填写自定义的模板名称和渲染函数。
render方法是核心绘制方法，其接收token列表和idx，获取当前token只需按照如下代码即可。在此示例中，展示了确定是opening tag和closing tag的方法，以及获取用户输入的配置项configs。configs格式为：Record<string, string[]>。如果用户没有填写配置项，或填写了错误的配置项，configs均不会有相关数据。

ts
import MarkdownItTemplate from 'markdown-it-template'
import type Token from 'markdown-it/lib/token.mjs'

md.use(templatePlugin, 'template-name-1', {
  configKeys: ['type'],
  render: function (tokens: Token[], idx: number) {
    const token = tokens[idx]
    const colorDict: Record<string, string> = {
      default: 'transparent',
      warning: '#F56C6C',
      success: '#67C23A'
    }
    if (token.nesting === 1) {
      // opening tag
      const configs = token.meta.configs
      // 用户配置项
      console.log(configs)
      const colorType = (configs['type'] && configs['type'][0]) || 'default'
      const color = colorDict[colorType.trim()] || colorType.trim()
      const style = `background-color: ${color};`
      return `<span style="${style}">`
    } else {
      // closing tag
      return '</span>'
    }
  }
})

console.log(md.render('{{template-name-1|warning content|type=warning}}'))

// Output:
//
// {"type": ["warning"]}
//
// <span style="background-color: #F56C6C;">warning content</span>

console.log(md.render('{{template-name-1|warning content}}'))

// Output:
//
// {}
//
// <span style="background-color: transparent;">warning content</span>
md
{{template-name-1|warning content|type=warning}}

{{template-name-1|warning content}}

{{template-name-1|warning content|type=warning}}

{{template-name-1|warning content}}

# 多配置项与配置项多值

configKeys参数接收一个字符串数组，其内部参数为配置项对应的键值。当为插件配置多个配置项时，配置项之间通过|分隔。当为一个配置项赋多个值时，值与值之间用英文逗号,分隔。

ts
md.use(templatePlugin, 'template-name-2', {
  configKeys: ['chars', 'color'],
  render: function (tokens: Token[], idx: number) {
    const token = tokens[idx]
    if (token.nesting === 1) {
      // opening tag
      const configs = token.meta.configs
      console.log(configs)
      const charsConf = (configs['chars']) || []
      const color = (configs['color']) || ''
      const chars = charsConf.join('-')
      return `<span style="color: ${color}">${chars}`
    } else {
      // closing tag
      return '</span>'
    }
  }
})

console.log(md.render('{{template-name-2|content|chars=a,b,c,d,e|color=red}}'))

// Output:
//
// {"chars": ["a", "b", "c", "d", "e"], "color": ["red"]}
//
// <span style="color: red">a-b-c-d-econtent</span>
md
{{template-name-2|content|chars=a,b,c,d,e|color=red}}

{{template-name-2|content|chars=a,b,c,d,e|color=red}}

TIP

使用=赋值时不需要在意书写顺序

md
{{template-name-2|content|color=red|chars=a,b,c,d,e}}

{{template-name-2|content|color=red|chars=a,b,c,d,e}}

# 配置项赋值方式

配置项默认使用=赋值，设置useIndex = true后支持通过下标确定配置项名称进行赋值。

ts
md.use(templatePlugin, 'template-name-3', {
  configKeys: ['chars', 'color'],
  useIndex: true,
  render: function (tokens: Token[], idx: number) {
    // 同 template-name-2
  }
})

// 此处不需要使用chars=a,b,c,d,e|color=red
console.log(md.render('{{template-name-3|content|a,b,c,d,e|red}}'))

// Output:
//
// {"chars": ["a", "b", "c", "d", "e"], "color": ["red"]}
//
// <span style="color: red">a-b-c-d-econtent</span>
md
{{template-name-3|content|a,b,c,d,e|red}}

{{template-name-3|content|a,b,c,d,e|red}}

直接填写的赋值顺寻的按照configKeys进行，如果希望直接赋值，可以使用index=value进行。

md
{{template-name-3|content|1=red|0=a,b,c,d,e}}

{{template-name-3|content|1=red|0=a,b,c,d,e}}

# Tokenize解析位置

Tokenize解析位置代表继续使用Markdown解析的位置，位置划分如下：{{位置0|位置1|位置2|位置3|...}}。位置0代表模板名字，此位置无法被Tokenize解析。默认情况位置一是唯一被Tokenize解析的位置。Tokenize解析位置通过tokenizePos控制，其接收一个number[] | boolean参数，true代表全部被解析，false等价于[1]，不被解析的位置会尝试解析为配置项，成功后放入configs。

ts
md.use(templatePlugin, 'template-name-4', {
  configKeys: ['chars', 'color'],
  tokenizePos: [], // 全部禁止解析
  render: function (tokens: Token[], idx: number) {
    // 同 template-name-2
  }
})

console.log(md.render('{{template-name-2|chars=a,b,c,d,e|color=red}}'))

// Output:
//
// {"chars": ["a", "b", "c", "d", "e"], "color": ["red"]}
//
// <span style="color: red">a-b-c-d-e</span>
md
{{template-name-4|chars=a,b,c,d,e|color=red}}

{{template-name-4|chars=a,b,c,d,e|color=red}}

需要注意的是，如果tokenizePos解析位置超过一个，就需要编写renderDivision方法。

ts
md.use(templatePlugin, 'tran', {
  useIndex: true,
  configKeys: ['from', 'to'],
  render: function (tokens: Token[], idx: number) {
    const token = tokens[idx]
    if (token.nesting === 1) {
      const name = Math.floor(Math.random() * 1000000).toString()
      const configs = token.meta.configs
      const from = (configs['from'] && configs['from'][0]) || '原文:'
      const to = (configs['to'] && configs['to'][0]) || '译文:'
      const input = `<label data-from="${from}" data-to="${to}" class="md-template-translate__label"><input type="checkbox" name="${name}" hidden ></label>`
      return '<span class="md-template-translate">' + input + '<span>'
    } else {
      return '</span></span>'
    }
  },
  renderDivision: function (token: Token[], idx: number) {
    return '</span><span>'
  },
  tokenizePos: [1, 2]
})

# 自定义模板匹配标识符

模板默认使用{{标识模板开始、使用}}标识模板结束、使用|标识模板配置项分隔。可以通过markerBegin、markerEnd、markerDivision进行修改。

WARNING

非必要还是不去修改比较好

# 参数

  • name - 模板名称（必填），字符串或数组，用于模板识别，模板名字请不要重复
    可选参数:
属性名 说明 类型 默认值
useEqual 模板配置项是否启用 = 进行配置项赋值 boolean true
useIndex 模板配置项是否启用下标顺序判定 boolean false
configKeys 配置项的键值，便于数据读取 string[] []
render 自定义渲染函数 (tokens: Token[], idx: number, _options: Options, env: any, slf: Renderer) => string -
renderDivision 自定义分割渲染函数 (tokens: Token[], idx: number, _options: Options, env: any, slf: Renderer) => string -
markerBegin 模板起始匹配标识符 string {{
markerEnd 模板结束匹配标识符 string }}
markerDivision 模板配置项分隔匹配标识符 string |
